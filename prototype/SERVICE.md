# CheckReading 서비스 개요

> 프로토타입 작업 시 참조용. 서비스의 핵심 개념·용어·기능을 정리.

---

## 한 줄 정의

**CheckReading**은 6~17세 영어 초심자를 위한 영어 읽기 솔루션으로,  
모바일 카메라 시선추적 기술로 아이가 책을 제대로 읽고 있는지 실시간 감독하고,  
부모가 함께 모니터링할 수 있는 서비스다.

---

## 사용자 유형

| 유형 | 설명 | 주요 화면 |
|------|------|----------|
| **부모 (parent)** | 자녀 계정을 관리하고 독서 상태를 모니터링 | 프로필, 서점, 책장, 내 정보 |
| **학생 (student)** | 책을 읽고 점수를 쌓는 실제 사용자 | 서점, 책장, 뷰어, 내 정보 |

- 한 부모 계정에 자녀 프로필 여러 개 등록 가능
- 학생은 직접 로그인하거나 부모가 프로필을 선택해 진입
- `cr_user.type`으로 분기 (`'parent'` | `'student'`)

---

## 핵심 기능 개요

### 1. 책 읽기 (뷰어)

영어 원서를 뷰어에서 읽는다. 시선추적이 활성화되면 읽기 데이터가 실시간으로 수집된다.

**뷰어 진입 경로**
```
서점 책 상세 → [책 읽기]         → 1챕터부터
서점 책 상세 → TOC 챕터 클릭     → 해당 챕터부터
부모 프로필  → [같이 보기]       → 자녀 현재 위치부터 (모니터링 모드)
```

**뷰어 컨트롤러 (B 레이아웃)**
- 화면 터치/클릭 → 상단 바 + 하단 컨트롤러 토글 (5초 후 자동 숨김)
- 상단: 나가기 | 챕터명 | ⚙ 설정
- 하단 슬라이더: 진행률
- 하단 버튼: [이전챕터] [뒤로] [▶재생/⏸중지] [앞으로] [다음챕터] [⋮더보기]
- 더보기 패널 (위로 슬라이드업): 재생 속도 + 기능 토글
- 재생 시 속도에 비례한 자동 스크롤 (TTS 동기화 예정)

**뷰어 기능 토글 4종**

| 기능 | 아이콘 | 설명 |
|------|--------|------|
| 오디오 | 🔊 | TTS 음성 재생 (고품질 AI 음성) |
| 포인터 | 👆 | 현재 읽는 위치를 시각적으로 표시 |
| 끊어읽기 | ✂️ | 의미 단위(청크)로 텍스트 분절 표시 |
| 시선 | 👁 | 시선 추적 활성화 (캘리브레이션 필요) |

---

### 2. 시선 추적 (Eye Tracking)

별도 장비 없이 모바일 카메라로 시선을 추적하는 것이 핵심 기술.

**시선 상태 3종**

| 상태 | 색상 | 의미 |
|------|------|------|
| **Reading** | 🟢 emerald | 텍스트 라인을 순서대로 따라 읽는 중 |
| **Scanning** | 🟡 amber | 텍스트를 훑어보거나 빠르게 넘기는 중 |
| **Lost** | 🔴 red | 화면에서 시선이 벗어났거나 집중하지 않는 상태 |

세 상태의 비율(%)을 실시간으로 측정해 읽기 품질을 평가한다.  
**Lost 30% 초과** 시 경고 표시 (프로필 카드 테두리 amber, 부모에게 알림).

---

### 3. 캘리브레이션 (시선 보정)

시선추적 정확도를 높이기 위해 뷰어 진입 시마다 보정 절차를 거친다.

**진입 플로우**
```
뷰어 열기
  └─ 얼굴 확인      카메라에 얼굴이 잘 잡히는지 확인 (타원 가이드)
       └─ 1점 간이 캘리  화면 중앙 점을 3초 응시
            └─ 읽기 시작
```

**캘리브레이션 종류**

| 종류 | 점수 | 소요시간 | 언제 |
|------|------|----------|------|
| **1점 간이** | 1점 | ~3초 | 뷰어 열 때마다, 챕터 이동마다 |
| **5점 정밀** | 5점 | ~15초 | 정확도 저하 시 권고, 설정에서 수동 실행 |

**5점 보정 위치 순서**: 좌상 → 우상 → 중앙 → 좌하 → 우하

**정밀 보정 권고**: 읽기 중 시선 감지 품질이 낮아지면 상단에 amber 배너 표시
```
"시선 감지 품질이 낮아졌습니다"  [재보정]  [나중에]
```

### 4. 알림 설정

부모가 자녀 프로필의 설정 패널 → [알림 설정]에서 조건별 알림을 on/off하고 기준값을 지정한다.

| 그룹 | 항목 | 기본값 |
|------|------|--------|
| **읽기 이벤트** | 읽기 시작 알림 | ON |
| | 읽기 완료 알림 | ON |
| **중단 감지** | 읽기 중단 N분 이상 시 | ON, **5분** (3/5/10/15/20분 선택) |
| **시선 상태** | Reading 비율 N% 이하로 저하 시 | ON, **50%** (30~70% 선택) |
| | Lost 비율 N% 이상 급증 시 | ON, **30%** (20~50% 선택) |
| **목표 & 습관** | 오늘 목표 달성 시 칭찬 알림 | ON |
| | 저녁 리마인더 (미읽음 시) | ON, **20:00** (18~22시 선택) |
| | 주간 읽기 리포트 (일요일) | ON |
| **서비스** | 새 책 추천 알림 | OFF |
| | 레벨 업 알림 | ON |

값이 있는 항목(중단 시간, 시선 비율, 리마인더 시각)은 토글 ON 시 pill 버튼으로 값 선택.  
데이터 구조: `notifyGroups` 배열 → 각 group에 `items[]` (id, icon, label, desc, on, value, unit, options)

### 5. 읽기 점수 (Reading Score)

읽기 세션마다 점수가 쌓인다. 시선 상태·읽기 속도·완독 여부 등을 종합해 산출 (구체적 알고리즘은 별도).

**집계 단위**

| 항목 | 설명 |
|------|------|
| 오늘 점수 | 당일 누적 읽기 점수 |
| 이번 달 점수 | 월간 누적 |
| 올해 점수 | 연간 누적 |
| 총 점수 (totalScore) | 전체 누적 |

**프로필 카드의 오늘 현황**
- 읽기점수 / 읽은 시간(분) / 읽은 단어수 / 시선 상태 비율(%)

---

### 5. 레벨 & 등급

**레벨 (Lv.)**: 읽기 능력 수준. Lv.1(입문)~Lv.6+ 으로 올라간다.  
서점에서 책마다 권장 레벨이 표시되어 본인 수준에 맞는 책을 찾을 수 있다.

**등급 (Grade)**: 활동량·점수 기반 명예 등급.

| 등급 | 색상 | 아이콘 |
|------|------|--------|
| 브론즈 | 🟤 orange | 🏆 |
| 실버 | ⚪ gray | 🏆 |
| 골드 | 🟡 yellow | 🏆 |
| 플래티넘 | 🔵 cyan | 🏆 |

---

### 6. 실시간 모니터링 (부모)

WebSocket으로 연결되어 자녀가 뷰어를 켜면 부모에게 실시간으로 상태가 전달된다.

**프로필 카드 온라인 상태**
- 아바타 우측하단 녹색 점 + "● 읽는 중" 배지
- 오늘 읽기 현황 지표가 1초마다 업데이트
- [같이 보기]: 부모가 자녀의 뷰어를 같은 화면으로 열어 함께 모니터링
- [얼굴 보기]: 자녀 기기 카메라 영상을 부모 화면에 실시간 표시 (집중도 육안 확인)

**프로필 카드 경고 표시**
- Lost 30% 초과 → 카드 테두리 `ring-amber-400`

### 7. 내 정보 (My Info)

`myinfo.html` — `isParent` 플래그로 학생/부모 뷰를 한 파일에서 분기.

**학생 뷰**
- 그라디언트 프로필 카드 (이름·나이·학년·성별·레벨·총점)
- 뱃지 스트립 (가로 스크롤)
- 읽기 점수 3종 (누적/올해/이번달)
- 월별 활동 캘린더 + 일별 읽기내역 슬라이드업 모달
- `profile-detail.html`과 동일 구조, 톱니바퀴(수정) 버튼 없음

**부모 뷰**
- 프로필 카드: 닉네임 + 부모 계정 뱃지 + 연동 구글 계정 이메일 + 편집 버튼
- 편집 시트 (슬라이드업): 프로필 사진 변경 + 닉네임 수정
- 구독 섹션: 현재 플랜(월간 10,000원) + **구독중단 버튼** (확인 모달) + 연간 전환 제안
- 결제 수단: 등록 카드 + **케밥 메뉴** (기본 결제수단 설정 / 결제수단 삭제) + 카드 추가 버튼
- 결제 내역 목록
- 기타 섹션: 이용약관 / 개인정보처리방침 (88dvh 슬라이드업 시트) / 로그아웃 / 회원탈퇴 (확인 모달)

```js
// 부모 뷰 Alpine data 주요 상태
cardMenuOpen: false,      // 결제수단 케밥 드롭다운
showCancelConfirm: false, // 구독중단 확인 모달
showTerms: false,         // 이용약관 시트
showPrivacy: false,       // 개인정보처리방침 시트
showLogoutConfirm: false, // 로그아웃 확인 모달
showDeleteConfirm: false, // 회원탈퇴 확인 모달
```

### 8. 책장 선택 삭제 모드 (Bookshelf Select Mode)

`bookshelf.html` — 책 선택 삭제 UX.

**진입 방법**
- **모바일**: 책 카드 롱프레스(500ms) → 진동 피드백 + 선택 모드 진입
- **웹**: 카드 호버 → 좌상단 원형 체크박스 노출 → 체크박스 클릭으로 선택 모드 진입

**선택 모드 UI**
- 헤더: "취소" | "N권 선택됨" | "전체선택/해제" (기존 헤더 교체)
- 하단: 빨간 "N권 삭제" 버튼 (탭바 교체)
- 선택된 카드: 인디고 테두리 ring + 반투명 오버레이

**삭제 동작**: 세션 내 `removedIds[]` 배열 필터 (localStorage 미반영)

### 9. 프로필 상세 차트 (Profile Detail Chart)

`profile-detail.html` — 기존 읽기점수 3카드 → Chart.js 탭 차트로 대체.

**탭 3종**

| 탭 | 차트 유형 | 내용 |
|----|-----------|------|
| 읽기 점수 | Line | 기간별 누적 읽기 점수 추이 |
| 시선 상태 | Stacked Bar | Reading / Scanning / Lost 비율(%) |
| 시선 집중도 | Line | 재생 포인트 대비 시선 일치율(%) |

**기간 선택**: 1주 / 1달 / 3달 pill 버튼  
**이전/다음 탐색**: ◀ [기간 라벨] ▶ 네비게이션 + 모바일 스와이프

**기술 주의사항**: Chart.js 인스턴스를 Alpine.js 반응형 data 객체에 저장하면 Alpine Proxy가 Chart 내부를 래핑해 렌더링 오류 발생.  
→ `let chartInst = null`을 `app()` 클로저 변수로 선언 (`return {}` 바깥), Alpine 반응형 밖에서 관리.

### 10. 서점 (Bookstore)

**선반 구조**
- **공통 선반**: 또래 인기, 이번 주 인기, 수준별 추천 등
- **프로필별 추천 선반**: 자녀별 선호 장르 기반 추천 (헤더에 원형 아바타 표시)

**책 표지 찜 표시**  
책 표지 우측상단에 프로필별 찜 하트 아이콘. 프로필 색상으로 구분.
- 홍길동(보라 `#667eea`) / 성춘향(핑크 `#f093fb`)

**책 상세 패널**  
슬라이드업으로 열림. 목차(TOC) / 후기 / 비슷한 책 탭 구성.  
후기 작성자 → 프로필 보기 → `profile-detail.html` (isOther=true)

### 11. 즐겨찾는 프로필

부모가 다른 학생의 독서 활동을 참고할 수 있도록 즐겨찾기 기능 제공.  
즐겨찾는 프로필의 `profile-detail.html`에서는 하트 아이콘(즐겨찾기 추가/제거)만 노출,  
톱니(설정) 버튼은 본인 자녀 프로필에서만 표시.

```js
// isOther: 다른 사용자의 자녀 프로필
profile.isOther === true  → 헤더: 하트 아이콘
profile.isOther === false → 헤더: 톱니 아이콘 (부모만)
```

---

### 12. 청크 읽기 (Chunked Reading)

의미 단위(청크)로 텍스트를 끊어 표시해 읽기 이해력을 향상시키는 기능.  
예: `"I didn't want"` / `"to be a half-blood."` 처럼 구 단위로 분절.  
뷰어의 ✂️ 끊어읽기 토글로 on/off.

## 프로필 데이터 구조

```js
profile = {
  id: 'hong',
  name: '홍길동',
  initial: '홍',            // 아바타에 표시되는 첫 글자
  age: 9,
  grade: '초3',             // 학년
  gender: '남',
  level: 'Lv.3',            // 읽기 레벨
  gradeLabel: '골드',        // 명예 등급
  favorites: 18,            // 찜한 책 수
  totalScore: 3240,         // 누적 읽기 점수
  color: 'linear-gradient(135deg,#667eea,#764ba2)',  // 프로필 그라디언트
  accent: '#667eea',        // 버튼 등에 쓰는 단색 (그라디언트 첫 색)
  online: true,             // 현재 뷰어 열림 여부 (WebSocket)
  today: {
    score: 124,             // 오늘 읽기 점수
    time: 42,               // 오늘 읽은 시간 (분)
    words: 1840,            // 오늘 읽은 단어 수
    eye: {
      reading: 65,          // Reading 비율 (%)
      scanning: 22,         // Scanning 비율 (%)
      lost: 13              // Lost 비율 (%) — 셋의 합 = 100
    }
  },
  isOther: false            // true: 즐겨찾는 다른 사용자의 자녀
}
```

## 화면 간 이동 흐름

```
index.html (로그인)
  ├─ 부모 → profiles.html
  │           ├─ 프로필 카드 클릭 → profile-detail.html
  │           ├─ [같이 보기]     → reader.html (모니터링 모드, 미구현)
  │           └─ [얼굴 보기]     → 카메라 모달 (profiles.html 내)
  │
  └─ 학생 → bookstore.html
              ├─ 책 상세 [책 읽기]    → reader.html
              ├─ 책 상세 TOC 챕터    → reader.html (해당 챕터)
              └─ 후기 프로필 보기    → profile-detail.html (isOther=true)

reader.html
  └─ 뒤로가기 → bookstore.html (또는 history.back)

공통 하단탭/사이드바
  프로필 → 소통 → 서점 → 내 책장 → 내 정보
  (profiles.html은 parent만 표시)
```

## localStorage 키 목록

| 키 | 내용 | 쓰는 곳 | 읽는 곳 |
|----|------|---------|---------|
| `cr_user` | 로그인한 사용자 `{name, type, initial}` | index.html | 모든 페이지 |
| `cr_profile` | 선택된 자녀 프로필 (profile 객체 전체) | profiles.html | profile-detail.html, reader.html |
| `cr_reader_chapter` | 뷰어 진입 챕터 번호 (0-based) | bookstore.html | reader.html |

## 기술 스택 (프로토타입)

| 항목 | 내용 |
|------|------|
| UI 프레임워크 | Alpine.js v3 (CDN) |
| CSS | Tailwind CSS v3 (CDN) |
| 빌드 | 없음 — HTML 파일 직접 서빙 |
| 저장소 | localStorage (서버 없음) |
| 카메라 | `navigator.mediaDevices.getUserMedia` |
| WebSocket | setInterval 시뮬레이션 (실제 연동 예정) |
| 시선추적 | 캘리브레이션 UI 구현, SDK 연동 예정 |
| 차트 | Chart.js 4.4.0 UMD CDN (Alpine 이전에 동기 로드) |
| 저장소(git) | `robotkiwi2/CheckReading-manual`, branch `main`, `/prototype/` |
